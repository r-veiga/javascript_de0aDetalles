Las dos pasadas al código de Javascript

Javascript hace dos pasadas al código.
Una primera para localizar cuáles son las variables y reservar una referencia en memoria.
Otra en ejecución donde asigna los valores a las referencias reservadas.

Por eso, emplear una variable declarada en un punto posterior a su ejecución no devuelve "ReferenceError: xxx is not defined" sino que devuelve el valor undefined 

<script>
  console.log( n1 );  
  console.log( a );   
  var a = "Variable A";  
  var b = "Variable B";  
  var c = "Variable C";  
  var n1 = 1;
  var n2 = 2;
  console.log( b );    
  console.log( c );  
  console.log( n2 );  
<\script>

Devolverá:

  undefined
  undefined
  Variable B
  Variable C
  2

undefined no es lo mismo que null

undefined significa que la variable ha sido creada pero que aún no se le ha asignado ningún valor.
null es un valor de asignación. Se le puede asignar a una variable para representar que no contiene ningún valor. 

undefined === null 		// Devuelve FALSE
undefined == null  		// Devuelve TRUE
