s1 v07 - Javascript es asíncrono

En cuanto se cargue esta página

En cuanto se cargue esta página, se lanzará una función Javascript "imprimir()" que iterará 8.000 veces para imprimir la misma cadena.

Pero si existe Ajax, ¿cómo que no es asíncrono?

Si Javascript fuera realmente asíncrono, podría ejecutar dos funciones en paralelo.
Lo voy a demostrar mediante la función con el bucle larguísimo. Durante su ejecución puedo pulsar un botón que a su vez lanza otra función.
Como Javascript es síncrono, la función lanzada por el botón no comenzará a ejecutarse hasta que termine la función que estaba en marcha (la del bucle).
Si Javascript fuera asíncrono, podrían ejecutarse simultáneamente ambas funciones y cada una terminaría cuando tuviese que terminar. 

<script>
  function imprimir() {
    var inicio = new Date();
    for(var i=0; i<8000; i++) {
      console.log("Función imprimir. Esto es una iteración del bucle de 8000.");
    }
    var fin = new Date();
    console.log("imprimir() tiempo de ejecución es  ==> " + (fin - inicio) );
    console.log("imprimir() hora de fin             ==> " + fin);
  }
  
  function clickBoton() {
    console.log("clickBoton() comienzo de ejecución ==> " + new Date() );
  }
  
  imprimir();     
</script>
